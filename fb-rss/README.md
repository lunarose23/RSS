# FB RSS — Facebook Page → RSS Feed

Self-hosted tool that scrapes public Facebook pages every 30 minutes and serves them as RSS feeds. Includes an admin UI to add/remove/pause pages. Runs on Render.com with HTTP Basic Auth toggled from the Render dashboard.

---

## What it does

- Scrapes public Facebook pages using a headless browser (Playwright)
- Caches post images locally so they don't expire
- Generates RSS XML files at `/feeds/<pagename>.xml`
- Keeps only the last 7 days of posts
- Admin UI at `/` to add, pause, force-scrape, and remove pages
- Scrapes automatically every 30 minutes

---

## Deploy to Render

### 1. Push to GitHub

```bash
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/YOURUSERNAME/fb-rss.git
git push -u origin main
```

### 2. Create a new Web Service on Render

1. Go to https://render.com → **New** → **Web Service**
2. Connect your GitHub repo
3. Render will auto-detect `render.yaml` — confirm the settings

### 3. Set environment variables in Render dashboard

Go to your service → **Environment** tab and set:

| Key | Value |
|-----|-------|
| `ADMIN_PASSWORD` | Your chosen password (never commit this) |
| `BASE_URL` | `https://your-app-name.onrender.com` |

`SECRET_KEY` is auto-generated by render.yaml.

### 4. Enable Persistent Disk

In Render dashboard → your service → **Disks** → the `fb-rss-data` disk should be created automatically from render.yaml. If not, add it manually:
- Mount path: `/data`
- Size: 1 GB

> ⚠️ Without the persistent disk, your database and images reset on every deploy. The Starter plan ($7/mo) is required for persistent disk on Render.

### 5. Enable HTTP Basic Auth (the password toggle)

1. Render dashboard → your service → **Settings** → scroll to **HTTP Basic Auth**
2. Toggle it **ON**, set a username and password
3. To turn it off: toggle **OFF** — takes effect immediately, no redeploy

> Your RSS reader (NetNewsWire, Reeder, etc.) will prompt for these credentials when you add a feed URL. Enter them once and they're saved.

## Using the Admin UI

Navigate to your app URL (e.g. `https://fb-rss.onrender.com`).

**Add a page:**
- Paste either the full URL (`https://www.facebook.com/pagename`) or just the page name (`pagename`)
- Enter your `ADMIN_PASSWORD`
- Click **Add Page** — it starts scraping immediately

**Feed URL:**
- Each page gets a URL like: `https://fb-rss.onrender.com/feeds/pagename.xml`
- Click ⎘ to copy it to clipboard
- Add this URL to any RSS reader on your phone

**Per-page controls:**
- **↻ Scrape now** — trigger an immediate scrape
- **⏸ Pause / ▶ Resume** — pause stops scheduled scrapes (feed stays available)
- **✕** — removes page and all its posts

**Remember password:**
- Click **Remember** after typing your password to save it in browser localStorage
- Saves you typing it every time

## RSS Feed format

Each post entry contains:
- A **View on Facebook →** link at the top
- Full post text
- All images (downloaded and served from your own server, so they won't expire)
- Each image is also linked back to the original Facebook post

## Selector maintenance

Facebook changes its DOM structure periodically. If scraping stops returning posts, the selectors in `scraper.py` need updating. Look for these sections:

```python
# CSS selectors — Facebook changes these periodically, update as needed
POST_CONTAINER_SELECTORS = [...]
```

Check the browser's DevTools on a Facebook page to find current selectors for:
- Post containers
- Post text elements
- Timestamp/post URL links
- Images

## Notes & known limitations

- **Facebook actively fights scrapers.** Selectors break, headless browsers get detected. This works but needs occasional maintenance.
- **Free Render tier** spins down after inactivity — upgrade to Starter ($7/mo) for always-on + persistent disk.
- **Images:** Downloaded at scrape time and served from your server. Should remain valid as long as your server is up.
- **Private pages** cannot be scraped — public pages only.
- **Post deduplication** is handled by post URL. If Facebook changes URL formats, you may see duplicates — clear the `posts` table to reset.
